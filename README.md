# Indirect Prompt Injections via Network Protocol Fields against LLM-Integrated IDS/IPS Systems

## Introduction

This project empirically evaluates whether indirect prompt injection attacks, embedded in HTTP header fields, can cause LLM-powered Intrusion Detection Systems (IDS) to misclassify malicious network traffic. Using a modified version of the CSIC 2010 dataset, we construct benign and malicious HTTP traffic samples augmented with a range of prompt injection payloads, and evaluate classification accuracy across multiple LLMs configured to behave as IDS systems. Metrics include attack success rate (ASR), false positive rate (FPR), accuracy and evasion deltas. We also explore whether defensive prompting strategies can mitigate these vulnerabilities.

## Dataset Augmentation Pipeline

### Step 1: Obtain the Dataset

Unzip `CSIC2010.zip` and note the path to the database CSV.

**Default path:** `data/csic_database.csv`  
Do **not** commit or push the unzipped dataset to the repository.

---

### Step 2: Preprocess the Dataset

Run the preprocessing tool to produce a balanced sample of benign and malicious HTTP requests, with malicious HTTP requests classified by attack type.

```
python -m augmentation.preprocess <raw_csv_path> <preprocessed_csv_path> <sample_size>
```

- `raw_csv_path` - path to the raw dataset CSV from Step 1 (default: `data/csic_database.csv`)
- `preprocessed_csv_path` - output path for the preprocessed CSV (default: `data/preprocessed.csv`)
- `sample_size` - number of benign samples and number of malicious samples to include, distributed evenly across attack types (default: `200`)

---

### Step 3: Augment the Dataset

Run the augmentation tool to generate the three-partition dataset used for evaluation.

```
python -m augmentation.augment <preprocessed_csv_path> <augmented_dataset_directory>
```

- `preprocessed_csv_path` - path to the preprocessed CSV from Step 2 (default: `data/preprocessed.csv`)
- `augmented_dataset_directory` - directory to store the three output CSV files (default: `data/augmented`)

Output files in `augmented_dataset_directory`:
- `A_benign.csv` - benign HTTP requests
- `B_malicious_clean.csv` - malicious HTTP requests without prompt injections
- `C_malicious_injected.csv` - malicious HTTP requests with prompt injection payloads


## Model Evaluation Pipeline

The evaluation pipeline is a Python script that calls the LLMs and parses their responses. It is run with the following command:

```
python evaluate.py <data_directory> <output_csv_path>
```

- `data_directory` - directory containing the three CSV files from Step 3 in the dataset augmentation pipeline (default: `data/augmented`)
- `output_csv_path` - path to the output CSV file (default: `results.csv`)

The output CSV file contains the following columns:
- `condition` - condition name (A_benign, B_clean, C_injected)
- `method` - HTTP method (GET, POST, etc.)
- `url` - URL
- `protocol` - HTTP protocol 
- `headers` - HTTP headers
- `body` - HTTP body
- `label` - label (benign or malicious)
- `attack_type` - attack type
- `injection_id` - ID of the injection (if applicable)
- `injection_text` - text of the injection (if applicable)
- `model` - LLM model used
- `prompt_version` - prompt version used

Note: The evaluation script requries the OpenAI, Anthropic, and Google API keys to be set in the `.env` file, and has additional module dependences: `python-dotenv`, `openai`, `anthropic`, and `google-generativeai`.

## Metrics, Analysis, and Artifacts Pipeline

The scripts `metrics.py` and `analysis.py` are used to generate metrics and visualizations for our interval review. The final artifacts for the paper are generated by the script `paper_artifacts.py`. 
The final paper is added as a PDF in the current directory: `NetworksCapstoneFinalPaper.pdf`.
